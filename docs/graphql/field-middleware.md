### 字段中间件

> 警告 **警告** 本章仅适用于代码优先方法。

字段中间件允许您在字段被解析前或后执行任意代码。字段中间件可以用来转换字段结果、验证字段参数或甚至检查字段级别的角色（例如，要求访问某个目标字段以便执行中间件函数）。

您可以将多个中间件函数连接到字段。这样，他们将按顺序在链中执行，其中前一个中间件决定是否调用下一个中间件。中间件函数在数组中的顺序非常重要。第一个解析器是外层最外层的，所以它将首先执行并最后执行（与 `@nestjs/package` 类似）。第二个解析器是第二外层的，所以它将第二次执行并第二次到最后执行。

#### 开始

让我们从创建一个简单的中间件开始，该中间件将在将字段值发送回客户端之前记录字段值：

```typescript
@ApiSecurity('basic')
@Controller('cats')
export class CatsController {}

```

> 提示 **提示** `next` 是一个对象，它包含与通常由 GraphQL 解析器函数接收的相同参数，而 `next` 是一个函数，让您执行下一个中间件在栈中或实际字段解析器。

> 警告 **警告** 字段中间件函数不能注入依赖项，也不能访问 Nest 的 DI 容器，因为它们旨在非常轻量级，不应该执行任何可能耗时的操作（例如，从数据库中检索数据）。如果您需要调用外部服务/查询数据源，请在 Guard/拦截器中将其绑定到根查询/ mutation 处理程序，并将其分配给 `object` 对象，这样您可以在字段中间件中访问（特定地，从 `object` 对象中）。

注意，字段中间件必须满足 `interface` 接口。在上面的示例中，我们首先执行 `fn` 函数（执行实际字段解析器并返回字段值），然后，我们将该值记录到我们的终端。也就是说，我们不想执行任何更改，所以我们简单地返回原始值。

现在，我们可以在 `@Decorator` 中注册我们的中间件，如下所示：

```typescript
const options = new DocumentBuilder().addSecurity('basic', {
  type: 'http',
  scheme: 'basic',
});

```

现在，每当我们请求 ``DocumentBuilder`` 字段时，原始字段值将被记录到控制台。

> 提示 **提示** 了解如何使用 `LINK_19` 功能实现字段级别的权限系统，请查看这个 `LINK_20`。

> 警告 **警告** 字段中间件只能应用于 ``DocumentBuilder`` 类。了解更多信息，请查看这个 `LINK_21`。

此外，如前所述，我们可以在中间件函数中控制字段的值。为了示例目的，让我们将recipe的标题大写（如果存在）：

```typescript
@ApiBasicAuth()
@Controller('cats')
export class CatsController {}

```

在这种情况下，每个标题将自动大写，当请求时。

类似地，您可以将中间件绑定到自定义字段解析器（一个带有 `@Decorator` 装饰器的方法），如下所示：

```typescript
const options = new DocumentBuilder().addBasicAuth();

```

> 警告 **警告** 如果在字段解析器级别启用了增强器（`LINK_22`），那么字段中间件函数将在任何拦截器、守卫等中执行（但是在方法上绑定的根级别增强器之前）。

#### 全局字段中间件

除了将中间件直接绑定到特定字段外，您还可以注册一个或多个中间件函数以全局注册。这样，它们将自动连接到您的对象类型中的所有字段。

```typescript
@ApiBearerAuth()
@Controller('cats')
export class CatsController {}

```

> 提示 **提示** 全局注册的字段中间件函数将在本地注册的中间件（那些直接绑定到特定字段）之前执行。
