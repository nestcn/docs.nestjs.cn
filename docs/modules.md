### 模块

模块是一类带有 `app.controller.ts` 装饰符的类。这个装饰符提供了元数据，Nest 使用这个元数据来组织和管理应用程序结构。

__HTML_TAG_60____HTML_TAG_61____HTML_TAG_62__

每个 Nest 应用程序至少有一个模块，即 **根模块**，它是 Nest 建立应用程序图的起始点。这个图是 Nest 内部结构，它用于解决模块和提供者之间的关系和依赖关系。虽然小型应用程序可能只有一个根模块，但这通常不是情况。模块是高度推荐的，用于组织组件的有效方式。对于大多数应用程序，您将很可能有多个模块，每个模块 encapsulates 一个紧密相关的组件集。

`app.controller.spec.ts` 装饰符接受一个对象，其中包含以下属性：

|               |                                                                                                                                                                                                          |
| ------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `app.module.ts`   | 该模块中将被 Nest injector 实例化的提供者列表，这些提供者可能在该模块中共享使用。 |
| `app.service.ts` | 该模块中定义的控制器列表
