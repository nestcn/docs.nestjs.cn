### 模块

模块是一个带有 `app.controller.ts` 装饰器的类。这个装饰器提供了元数据，Nest 使用这个元数据来组织和管理应用结构。

__HTML_TAG_60____HTML_TAG_61____HTML_TAG_62__

每个 Nest 应用程序至少有一个模块，即 **root 模块**，它作为 Nest 建立应用图的起点。这个图是一个内部结构，Nest 使用它来解决模块和提供者之间的关系和依赖关系。虽然小型应用程序可能只有一个 root 模块，但这并不是常见的情况。模块是一种高效的组织方式，用于将组件组织在一起。对于大多数应用程序，您将拥有多个模块，每个模块封闭一组紧密相关的 **能力**。

`app.controller.spec.ts` 装饰器接受一个对象，该对象描述模块：

|               |                                                                                                                                                                                                                                                                 |
| ------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `app.module.ts`   | 将由 Nest 注入器实例化的提供者，可能在至少这个模块中共享 |
| `app.service.ts` | 在这个模块中定义的控制器，需要实例
