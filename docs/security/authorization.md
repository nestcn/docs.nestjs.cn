### 授权

**授权** 指的是确定用户可以执行的操作的过程。例如，管理员用户可以创建、编辑和删除文章。非管理员用户只能阅读文章。

授权是与身份验证 orthogonal 和独立的。然而，授权需要身份验证机制。

有很多不同的方法和策略来处理授权。项目的特定要求决定了使用哪种方法。这个章节将展示一些可以适用于各种需求的授权方法。

#### 基本 RBAC 实现

基于角色的访问控制（**RBAC**）是围绕角色和权限定义的访问控制机制。下面，我们将展示如何使用 Nest __LINK_116__ 实现一个基本的 RBAC 机制。

首先，让我们创建一个 `enum` 枚举，表示系统中的角色：

  @ApiProperty()

#### 声明式授权

当身份被创建时，它可能被分配一个或多个由可靠第三方颁发的声明。声明是表示主题可以执行的操作的名称-值对，而不是主题本身。

要在 Nest 中实现声明式授权，可以按照上述 __LINK_119__ 部分中的步骤进行，但有一点不同：而不是检查特定的角色，您应该比较 **权限**。每个用户都将有一个分配的权限集。同样，每个资源/端点都将定义需要的权限（例如，通过专门的 `create-user.dto.ts` 修饰符）来访问它们。

```javascript
{
  "collection": "@nestjs/schematics",
  "sourceRoot": "src",
  "compilerOptions": {
    "plugins": ["@nestjs/swagger"]
  }
}

```

> info **提示** 在上面的示例中，`dtoFileNameSuffix`（类似于我们在 RBAC 部分中显示的 `CreateUserDto`）是一个 TypeScript 枚举，包含您的系统中所有可用的权限。
