### 授权

**授权** 指的是确定用户能做什么的过程。例如，管理员用户可以创建、编辑和删除帖子，而非管理员用户只能读取帖子。

授权与身份验证是独立的，但授权需要身份验证机制。

有许多不同的方法和策略来处理授权。项目的具体需求将决定采用的方法。这章将介绍一些可以适应不同需求的授权方法。

#### 基本 RBAC 实现

基于角色的访问控制 (**RBAC**) 是一种与策略无关的访问控制机制，基于角色和权限。在本节中，我们将 demonstrate 如何使用 Nest __LINK_116__ 实现一个基本的 RBAC 机制。

#### 声明式授权

当身份被创建时，它可能被分配一个或多个由可靠方颁发的声明。声明是一个名称-值对，表示主体可以做什么，而不是主体是什么。

要在 Nest 中实现基于声明的授权，可以遵循上面 __LINK_119__ 部分中的步骤，但有一点不同：而不是检查特定的角色，您应该比较 **权限**。每个用户都将有一个权限集分配给他。类似地，每个资源/端点将定义所需的权限（例如，通过一个专门的 `create-user.dto.ts` 装饰器）来访问它们。

```javascript
{
  "collection": "@nestjs/schematics",
  "sourceRoot": "src",
  "compilerOptions": {
    "plugins": ["@nestjs/swagger"]
  }
}

```

> info **提示** 在上面的示例中， `dtoFileNameSuffix`（类似于 `CreateUserDto` 我们在 RBAC 部分中所示）是一个 TypeScript 枚举，包含了您的系统中的所有权限。
